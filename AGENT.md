# Pet-care Project Notes

## Project Overview

This project is a Next.js App Router version of a pet grooming landing page. It was migrated from the original static `index.html`.

The current site is a single-page pet grooming storefront with these sections:

- Hero and navigation
- Services
- Care advantages
- Customer testimonials
- Booking form display
- Gallery
- Contact information

The project is configured for static export, so it can be opened from `out/index.html` without a local server.

## Directory Structure

```text
Pet-care/
|-- app/
|   |-- layout.js        # App Router root layout and metadata
|   |-- page.js          # Main page content and section data
|   `-- globals.css      # Global styling and responsive layout
|-- public/
|   `-- assets/          # Source images used by the Next app
|-- out/                 # Generated static export
|-- assets/              # Original image folder from the static HTML version
|-- index.html           # Original pre-Next static page backup
|-- next.config.js       # Next config; static export enabled
|-- package.json         # Project scripts and dependencies
|-- pnpm-lock.yaml       # Installed dependency lockfile
`-- AGENT.md             # This project guide
```

Generated folders:

- `.next/`: Next.js build cache and production build data.
- `node_modules/`: Installed dependencies.
- `.pnpm-store/`: Local pnpm package cache.

These generated folders are ignored by `.gitignore`.

## Main Scripts

Defined in `package.json`:

```bash
npm run dev
npm run build
npm run start
```

Usage:

- `npm run dev`: Start the Next.js development server.
- `npm run build`: Build the project and export static files into `out/`.
- `npm run start`: Start a production Next server. This is less important here because the project uses static export.

Note: in this Codex environment, system `npm` may not be available. The project has already been built using the bundled Node runtime and local dependencies.

## Offline Preview Workflow

For preview without a local server:

1. Edit source files under `app/` or `public/`.
2. Run a build.
3. Open or refresh `out/index.html`.

Browser URL:

```text
file:///C:/Users/Jialin%20Yue/Desktop/Pet-care/out/index.html
```

Static export is enabled in `next.config.js`:

```js
const nextConfig = {
  assetPrefix: "./",
  output: "export",
};
```

This lets `out/index.html` load CSS, JavaScript, and local images from disk.

Remote Unsplash images still require internet access. For fully offline visuals, replace remote image URLs in `app/page.js` with files under `public/assets/`.

## Development Notes

- Keep source edits in `app/` and `public/`.
- Keep `out/` as generated output only.
- Keep the original `index.html` and `assets/` as backup unless the user asks to remove them.
- The current page language is Simplified Chinese, but this guide is ASCII-only to avoid encoding problems.
- The booking form is display-only. It does not submit data.
- Navigation anchors currently include `#services`, `#why`, `#booking`, and `#contact`.

## Common Follow-Up Tasks

To update customer reviews:

1. Open `app/page.js`.
2. Edit the `testimonials` array.
3. Rebuild and refresh `out/index.html`.

To update service cards:

1. Open `app/page.js`.
2. Edit the `services` array.
3. Rebuild and refresh `out/index.html`.

To update colors, layout, or mobile behavior:

1. Open `app/globals.css`.
2. Edit the related section styles.
3. Rebuild and refresh `out/index.html`.

To replace local gallery images:

1. Put new image files in `public/assets/`.
2. Update the `gallery` array in `app/page.js`.
3. Rebuild and refresh `out/index.html`.
